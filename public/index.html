Підпис документів для VilnoCheck

Сервіс підписує документ локально на робочому місці та формує ZIP-пакет із оригіналом, відокремленим підписом і метаданими. Підтримуються три режими: апаратний токен через IIT browser integration, файловий ключ PrivatBank у форматі .jks

1. Метод підписання

Важливо: приватний ключ і пароль залишаються на робочому місці. Бекенд VilnoCheck отримує лише вже створений підпис та безпечні метадані сесії.

2. Підготовка клієнта

Для IIT / токена потрібно один раз налаштувати робоче місце:
  • Chrome або Edge
  • встановлене IIT browser extension
  • встановлені IIT web libraries / local agent
  • додайте цей сайт до trusted sites, якщо IIT попросить
  • встановлені драйвери / middleware для Crystal-1
Очікується перевірка IIT agent…
Для PrivatBank JKS режиму:
  • вистачає сучасного браузера з підтримкою web worker
  • оберіть локальний .jks контейнер у блоці нижче
  • пароль контейнера використовується лише локально в браузері
  • IIT extension / hardware token для цього режиму не потрібні
