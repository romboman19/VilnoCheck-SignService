# VilnoCheck SignService

Сервіс для підписання документів кваліфікованим електронним підписом (КЕП). Підтримує три методи підпису: IIT токен, PrivatBank JKS, Хмарний КЕП.

## Архітектура

**Ключова принцип:** Приватний ключ ніколи не виходить за межі браузера клієнта.

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Браузер   │────▶│ SignService │────▶│  ДПС (ПРРО) │
│  (main.js)  │     │  (server)   │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
       │
       ▼
┌─────────────┐
│ IIT Agent / │
│   КЕП в     │
│  хмарі КНЕДП│
└─────────────┘
```

- Браузер завантажує документ → сервер повертає `signingPayloadBase64`
- Клієнт підписує локально за допомогою IIT Agent або хмарного КЕП
- Підпис відправляється на сервер → сервер відправляє в ДПС
- Результат: підписаний документ + ZIP-пакет для завантаження

## Методи підпису

### 1. IIT Token (`iit-token`)

**Вимоги:**
- Встановлений [IIT Agent](https://iit.com.ua/downloads)
- Токен (JaCarta, ALM, AvPass тощо) підключений до ПК
- Браузер Chrome/Edge/Firefox

**Флоу:**
1. "Перевірити IIT agent" → перевірка з'єднання з Agent
2. "Оновити носії" → список токенів
3. Вибір ключа + PIN → "Зчитати ключ"
4. Підписання документа

### 2. PrivatBank JKS (`privatbank-jks`)

**Вимоги:**
- Файл `.jks` (Java KeyStore) від ПриватБанку
- Пароль до контейнера

**Флоу:**
1. Завантаження `.jks` файлу
2. Вибір ключа зі списку
3. Введення пароля → "Зчитати JKS ключ"
4. Підписання документа

### 3. Хмарний КЕП (`cloud-kep`)

**Вимоги:**
- `CLOUD_SIGN_ENABLED=1` у env
- Обліковий запис у хмарного КНЕДП (Дія, Вчасно, інші)
- Мобільний додаток КНЕДП для підтвердження

**Флоу:**
1. Вибір КНЕДП зі списку
2. Введення ідентифікатора (РНОКПП/телефон)
3. "Зчитати ключ" → запит на підтвердження в мобільному додатку
4. Підтвердження в додатку (можливо + 2FA код)
5. Ключ завантажено → підписання документа

## Змінні оточення

| Змінна | За замовчуванням | Опис |
|--------|------------------|------|
| `PORT` | `3017` | Порт сервера |
| `HOST` | `0.0.0.0` | Інтерфейс для прослуховування |
| `SIGN_STORAGE_DIR` | `./storage` | Директорія для тимчасових файлів |
| `CLOUD_SIGN_ENABLED` | (не встановлено) | `1` для увімкнення хмарного КЕП |
| `CLIENT_API_KEY` / `API_KEY` | (не встановлено) | API ключ для авторизації |
| `NODE_ENV` | — | `production` для production режиму |

## API Endpoints

### Public (без авторизації)

| Метод | Шлях | Опис |
|-------|------|------|
| GET | `/api/bootstrap` | Конфігурація: доступні методи, версія |
| GET | `/api/health` | Health check |
| GET | `/config.js` | Runtime конфігурація |

### Authenticated (require `X-API-Key` header)

| Метод | Шлях | Опис |
|-------|------|------|
| POST | `/api/documents` | Завантажити документ для підпису |
| PATCH | `/api/documents/:id/session` | Оновити сесію документа |
| POST | `/api/documents/:id/signature` | Надіслати підпис |
| GET | `/api/documents/:id/package` | Завантажити ZIP-пакет |

### Proxy

| Метод | Шлях | Опис |
|-------|------|------|
| ALL | `/pki/ProxyHandler` | Проксі для IIT PKI сервісів |

## Чекліст тестування

### IIT Token
- [ ] IIT Agent встановлено і запущено
- [ ] Токен підключений і видно у списку носіїв
- [ ] PIN введено правильно → ключ зчитано
- [ ] Документ підписано → отримано URL для завантаження

### PrivatBank JKS
- [ ] `.jks` файл завантажується
- [ ] Пароль правильний → список ключів відображається
- [ ] Ключ обрано → зчитано успішно
- [ ] Документ підписано → отримано URL для завантаження

### Хмарний КЕП
- [ ] `CLOUD_SIGN_ENABLED=1` встановлено
- [ ] `/api/bootstrap` повертає 3 методи
- [ ] UI: dropdown КНЕДП, поле ідентифікатора, кнопка "Зчитати ключ"
- [ ] Запит на підтвердження надходить у мобільний додаток
- [ ] Після підтвердження ключ завантажено
- [ ] (Опціонально) 2FA код запитується і приймається
- [ ] Документ підписано → отримано URL для завантаження

## Безпека

- Приватний ключ не зберігається на сервері
- Всі операції з ключем виконуються в браузері через IIT Agent або KSP
- Сервер отримує тільки готовий підпис у форматі CAdES-X-Long
- API вимагає `X-API-Key` header для всіх операцій з документами

## Ліцензування

- IIT Agent — вимагає окрему ліцензію від IIT
- Бібліотека `@it-enterprise/digital-signature` — власна ліцензія
